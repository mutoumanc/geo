> 下面的“涉密样本”均指 **模拟涉密/敏感场景样本**，用于测试数据识别、脱敏、权限控制、格式兼容性、数据治理流程等；所有名称、坐标、属性均应视为虚构，不对应任何真实敏感设施或真实业务系统。

由于你列出的格式很多，且部分格式属于商业/专有格式，直接为每一种格式完整生成 20 个真实可打开文件会非常庞大，也有些格式无法用开源脚本稳定创建。因此下面采用更实用的数据分析/数据治理方式：

1. 先给出 **10 个不涉密样本主题池**
2. 再给出 **10 个模拟涉密样本主题池**
3. 然后按 56 种格式逐一映射：每种格式对应 10 个不涉密样本 + 10 个模拟涉密样本
4. 最后给出一个脚本，用于批量生成目录、样本清单、元数据 JSON，后续可按格式继续扩展为真实文件

# 一、样本命名规则

建议统一命名：

```text
格式_样本类型_序号_主题.扩展名
```

例如：

```text
shp_public_01_city_poi.shp
shp_sensitive_sim_01_facility_boundary.shp

tif_public_01_landcover.tif
tif_sensitive_sim_01_restricted_area_dem.tif

las_public_01_urban_lidar.las
las_sensitive_sim_01_controlled_facility_pointcloud.las
```

# 六、生成 56 × 20 个样本清单的 Python 脚本

下面脚本不会生成真实涉密数据，只会生成：

```text
sample_manifest.csv
sample_metadata.json
每种格式的目录
每个样本的占位文件
```

你可以后续把占位文件替换成真实的不涉密测试数据或模拟数据。

```python
import os
import json
import pandas as pd

OUT_DIR = "format_sample_manifest"
os.makedirs(OUT_DIR, exist_ok=True)

formats = [
    "shp", "mdb", "gdb", "dwg", "dxf", "edb", "accdb", "e00", "vot",
    "udd", "udbx", "sdd", "wp", "wt", "w", "dgn", "tab", "mif", "kml", "kmz",
    "adf", "pg", "jpeg", "bmp", "png", "tif", "tiff", "bil", "bip", "bsq",
    "iq", "ecw", "sid", "hdf", "dem", "img", "grd", "dsm", "asc",
    "obj", "las", "ply", "pod", "osgb", "osg", "osgt", "3mx", "b3dm",
    "pts", "off", "xyz", "stl", "laz", "neu", "stp", "step"
]

public_samples = [
    ("01", "city_poi", "城市 POI 点位", "public"),
    ("02", "road_centerline", "道路中心线", "public"),
    ("03", "admin_boundary", "行政区边界", "public"),
    ("04", "park_greenland", "公园绿地范围", "public"),
    ("05", "bus_route", "公交线路", "public"),
    ("06", "river_network", "河流水系", "public"),
    ("07", "building_footprint", "建筑物轮廓", "public"),
    ("08", "landuse_patch", "土地利用斑块", "public"),
    ("09", "tour_route", "旅游路线", "public"),
    ("10", "sampling_point", "公开采样点", "public"),
]

sensitive_sim_samples = [
    ("01", "facility_boundary", "虚构受控设施边界", "sensitive_simulated"),
    ("02", "underground_pipeline", "虚构地下管线", "sensitive_simulated"),
    ("03", "patrol_route", "虚构安防巡逻路线", "sensitive_simulated"),
    ("04", "key_asset_point", "虚构重点资产点位", "sensitive_simulated"),
    ("05", "command_site", "虚构应急指挥点", "sensitive_simulated"),
    ("06", "communication_node", "虚构通信节点", "sensitive_simulated"),
    ("07", "surveillance_area", "虚构监控覆盖区", "sensitive_simulated"),
    ("08", "access_control_zone", "虚构访问控制分区", "sensitive_simulated"),
    ("09", "storage_area", "虚构物资仓储区", "sensitive_simulated"),
    ("10", "project_redline", "虚构敏感项目红线", "sensitive_simulated"),
]

records = []

for fmt in formats:
    fmt_dir = os.path.join(OUT_DIR, fmt)
    os.makedirs(fmt_dir, exist_ok=True)

    for seq, slug, cn_name, cls in public_samples:
        filename = f"{fmt}_public_{seq}_{slug}.{fmt}"
        filepath = os.path.join(fmt_dir, filename)

        with open(filepath, "w", encoding="utf-8") as f:
            f.write(
                f"PLACEHOLDER SAMPLE FILE\n"
                f"format={fmt}\n"
                f"classification=public\n"
                f"sample_id={seq}\n"
                f"sample_name={cn_name}\n"
                f"note=This is a non-sensitive placeholder file.\n"
            )

        records.append({
            "format": fmt,
            "classification": "public",
            "sample_id": seq,
            "sample_slug": slug,
            "sample_name_cn": cn_name,
            "filename": filename,
            "path": filepath
        })

    for seq, slug, cn_name, cls in sensitive_sim_samples:
        filename = f"{fmt}_sensitive_sim_{seq}_{slug}.{fmt}"
        filepath = os.path.join(fmt_dir, filename)

        with open(filepath, "w", encoding="utf-8") as f:
            f.write(
                f"PLACEHOLDER SIMULATED SENSITIVE SAMPLE FILE\n"
                f"format={fmt}\n"
                f"classification=sensitive_simulated\n"
                f"sample_id={seq}\n"
                f"sample_name={cn_name}\n"
                f"note=This is synthetic simulated sensitive data for testing only. "
                f"It does not represent any real classified data.\n"
            )

        records.append({
            "format": fmt,
            "classification": "sensitive_simulated",
            "sample_id": seq,
            "sample_slug": slug,
            "sample_name_cn": cn_name,
            "filename": filename,
            "path": filepath
        })

df = pd.DataFrame(records)

csv_path = os.path.join(OUT_DIR, "sample_manifest.csv")
json_path = os.path.join(OUT_DIR, "sample_metadata.json")

df.to_csv(csv_path, index=False, encoding="utf-8-sig")

with open(json_path, "w", encoding="utf-8") as f:
    json.dump(records, f, ensure_ascii=False, indent=2)

print(f"生成完成：{OUT_DIR}")
print(f"样本总数：{len(records)}")
print(f"清单文件：{csv_path}")
print(f"元数据文件：{json_path}")
```

运行：

```bash
python generate_sample_manifest.py
```

将生成：

```text
56 种格式 × 20 个样本 = 1120 个样本占位文件
```

# 七、推荐的分类标签字段

为了后续做数据治理、权限控制、自动识别，可以给每个样本加如下字段：

| 字段                  | 示例值                                                           | 说明         |
| --------------------- | ---------------------------------------------------------------- | ------------ |
| `format`            | `shp`                                                          | 文件格式     |
| `sample_id`         | `01`                                                           | 样本编号     |
| `classification`    | `public` / `sensitive_simulated`                             | 数据分类     |
| `data_type`         | `vector` / `raster` / `pointcloud` / `mesh` / `cad`    | 数据类型     |
| `geometry_type`     | `Point` / `LineString` / `Polygon` / `Raster` / `Mesh` | 空间类型     |
| `crs`               | `EPSG:4326`                                                    | 坐标系统     |
| `source_type`       | `synthetic`                                                    | 数据来源     |
| `is_real_sensitive` | `false`                                                        | 是否真实涉密 |
| `usage`             | `format_test` / `permission_test`                            | 使用目的     |
| `desensitized`      | `true`                                                         | 是否脱敏     |
| `owner`             | `test_lab`                                                     | 数据责任方   |
| `access_level`      | `open` / `restricted_test`                                   | 访问级别     |
